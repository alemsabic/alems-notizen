# Refactoring von Monster-Notizen

*Wenn eine Notiz zu groß wird oder mehrere Ideen enthält, muss sie systematisch in atomare Teile zerlegt werden.*

Atomizität ist ein Ziel, keine Startbedingung. Notizen wachsen über Zeit, oder es rutschen mehrere Konzepte hinein. Das ist normal. Entscheidend ist, diese Überlastungen zu erkennen und aufzulösen.

Monster-Notizen entstehen, wenn:
- Eine Notiz über 1000 Wörter wird
- Der Titel zu vage wird ("Gedanken zu X")
- "Und außerdem..." mehrfach auftaucht
- Links auf die Notiz mehrdeutig werden (welcher Teil ist gemeint?)
- Du selbst vergisst, was alles drin steht

Refactoring ist die Lösung: Man teilt die Notiz in atomare Zettel auf und erstellt eine Structure-Note, die zeigt, wie sie zusammenhängen. Wichtig ist der dritte Schritt: **Rückverweise aktualisieren**. Wenn Links auf die alte Notiz zeigten, müssen sie jetzt auf die neuen Teile zeigen – sonst entstehen tote Links.

Das ist keine Schwäche des Systems, sondern ein natürlicher Prozess. Ein Zettelkasten, der nicht refaktoriert wird, wird zur Ablage statt zur Denkmaschine.
